المتغيرات (Variables)

المتغير هو مكان في الذاكرة نخزن فيه قيمة ونستطيع استخدامها لاحقًا.

        
          name = "Mubarak"     # نص (String)
          age = 20             # رقم صحيح (Integer)
          height = 1.75        # رقم عشري (Float)
          is_student = True    # قيمة منطقية (Boolean)

          # طباعة القيم
          print("Name:", name)
          print("Age:", age)
          print("Height:", height)
          print("Student:", is_student)

(String) نص: نضعه بين علامات تنصيص " "

(Integer) رقم صحيح : أرقام بدون فاصلة عشرية مثل 20

(Float) رقم عشري: أرقام تحتوي على فاصلة مثل 1.75

(Boolean) قيمة منطقية: تحتمل فقط True أو False

أنواع البيانات (Data Types) 
        
            # أنواع البيانات الأساسية

          name = "Mubarak"      # String (نص)
          age = 20              # Integer (عدد صحيح)
          height = 1.75         # Float (عدد عشري)
          is_student = True     # Boolean (صح/خطأ)

          # طباعة نوع كل متغير
          print(type(name))
          print(type(age))
          print(type(height))
          print(type(is_student))

name هو نص String لأنه داخل علامات اقتباس

age عدد صحيح Integer بدون فاصلة، وheight عدد عشري Float

is_student قيمة منطقية Boolean تكون إما True أو False.

أوامر ()print تقوم بعرض القيم على الشاشة، ولعرض النوع نستخدم ()type

عمليات المقارنة Comparison
        
          a = 10
          b = 5

          print(a == b)  # False (هل متساويين؟)
          print(a != b)  # True (هل مختلفين؟)

          print(a < b)   # False
          print(a >= b)  # True

عمليات المقارنة في Python بين المتغيرين a و b

== تتحقق هل القيم متساوية، و != هل القيم مختلفة

> و < تستخدم لمعرفة الأكبر والأصغر

>= و <= تعني أكبر أو يساوي وأصغر أو يساوي

كل النتائج تكون True أو False ويتم عرضها باستخدام ()print

العمليات المنطقية Logical
        
         x = True
         y = False

         print(x and y)  # False
         print(x or y)   # True
         print(not x)    # False

العمليات المنطقية في Python باستخدام قيم True و False

and ترجع True فقط إذا كانت القيمتان True، وهنا النتيجة False

or ترجع True إذا كانت واحدة على الأقل True، وهنا النتيجة True

not تعكس القيمة، فـ True تصبح False

تُستخدم هذه العمليات كثيرًا في الشروط if واتخاذ القرارات في البرامج

الجمل الشرطية (If / Else / Elif)

if فقط

ينفذ فقط إذا الشرط صحيح

        
         num = 10

         if num > 5:
            print("Number is greater than 5")

هذا الكود يوضح استخدام جملة الشرط if في Python

يتم تخزين القيمة 10 في المتغير num

الشرط num > 5 يتحقق إذا كان العدد أكبر من 5

إذا كان الشرط صحيح True، يتم تنفيذ الأمر داخل if

في هذه الحالة سيتم طباعة "Number is greater than 5"

if + else

إما هذا أو هذا

        
          num = 3

          if num > 5:
              print("Big number")
          else:
              print("Small number")

هذا الكود يوضح استخدام if و else في Python

يتم تخزين القيمة 3 في المتغير num

الشرط num > 5 يتم التحقق منه

إذا كان الشرط صحيح يُطبع "Big number"، وإذا كان خطأ يُنفذ else

بما أن 3 أصغر من 5، سيتم طباعة "Small number"

if + elif + else

تستخدم elif لعدة شروط

        
        num = 10

        if num > 10:
            print("Greater than 10")
        elif num == 10:
            print("Equal to 10")
        else:
            print("Less than 10")

هذا الكود يوضح استخدام if و elif و else في Python

يتم التحقق أولًا هل num > 10، وإذا لم يتحقق ينتقل إلى الشرط التالي

elif تستخدم لفحص شرط آخر مثل num == 10.

إذا لم يتحقق أي شرط، يتم تنفيذ else

بما أن القيمة 10، سيتم طباعة "Equal to 10"

الجمل الشرطية (If / Else / Elif)

        
          age = int(input("Enter your age: "))

          if age < 13:
              print("Child")
          elif age < 18:
              print("Teen")
          else:
              print("Adult")

هذا الكود يطلب من المستخدم إدخال عمره ثم يحوله إلى عدد باستخدام ()int

يتم استخدام if للتحقق إذا كان العمر أقل من 13 (Child)

elif تتحقق إذا كان العمر أقل من 18 (Teen).

إذا لم يتحقق أي من الشرطين، يتم تنفيذ else (Adult)

الكود يصنّف الشخص حسب عمره إلى طفل أو مراهق أو بالغ

الحلقات (Loops - For / While)

الحلقات تستخدم لتكرار تنفيذ الكود أكثر من مرة بدون ما تعيد كتابته

حلقة for

range(5) يبدأ من 0 إلى 4 ,i متغير يتغير كل مرة

        
        for i in range(5):
            print(i)

هذا الكود يوضح استخدام حلقة for في Python

range(5) تُنشئ أرقام من 0 إلى 4 (لا تشمل 5)

المتغير i يأخذ قيمة جديدة في كل تكرار

في كل مرة يتم تنفيذ print(i)

النتيجة ستكون طباعة الأرقام من 0 إلى 4

حلقة while

تستمر الحلقة طالما الشرط صحيح يجب زيادة i لكي لا تصبح حلقة لا نهائية

        
         i = 0

         while i < 5:
             print(i)
             i += 1

هذا الكود يوضح استخدام حلقة while في Python

يبدأ المتغير i بالقيمة 0.

تستمر الحلقة طالما الشرط i < 5 صحيح

في كل مرة يتم طباعة i ثم زيادته بـ 1

النتيجة هي طباعة الأرقام من 0 إلى 4

خطأ حلقة لا نهائية

هنا البرنامج لن يتوقف

        
        i = 0

        while i < 5:
            print(i)

هذا الكود يحتوي على حلقة while بدون زيادة المتغير

يبدأ i بالقيمة 0 والشرط i < 5 دائمًا صحيح

لأنه لا يوجد i += 1، لن تتغير قيمة i

بالتالي ستستمر الحلقة إلى ما لا نهاية Infinite Loop

وسيتم طباعة الرقم 0 بشكل متكرر

التحكم في الحلقات (Break / Continue / Pass)

نستخدم هذه الأوامر داخل الحلقات for أو while للتحكم في التنفيذ.

break إيقاف الحلقة

يوقف الحلقة فورًا حتى لو لم تنتهِ

        
         for i in range(10):
             if i == 5:
                 break
              print(i)

هذا الكود يوضح استخدام break داخل حلقة for في Python

الحلقة range(10) تطبع الأرقام من 0 إلى 9

عندما تصل القيمة i إلى 5، يتم تنفيذ break

break تقوم بإيقاف الحلقة فورًا

لذلك سيتم طباعة الأرقام من 0 إلى 4 فقط

تخطي continue

يتخطى التكرار الحالي ويكمل الباقي

تم تخطي الرقم 2

        
          for i in range(5):
              if i == 2:
                  continue
              print(i)

هذا الكود يوضح استخدام continue داخل حلقة for في Python

الحلقة تطبع الأرقام من 0 إلى 4

عندما تكون قيمة i تساوي 2 يتم تنفيذ continue

continue تتخطى هذه الدورة ولا تنفذ print(i)

لذلك سيتم طباعة: 0، 1، 3، 4 (بدون 2)

تجاهل مؤقت pass

لا يفعل شيء (مكان فارغ للكود)

البرنامج يعمل بدون خطأ

        
          for i in range(5):
              if i == 2:
                  pass
              print(i)

هذا الكود يوضح استخدام pass داخل حلقة for في Python

pass تعني لا تفعل شيئًا تُستخدم كعنصر فارغ

عندما تكون i == 2، لا يحدث أي تغيير ويستمر التنفيذ

بالتالي يتم تنفيذ print(i) في كل مرة

النتيجة: طباعة الأرقام من 0 إلى 4 بدون تغيير

continue و break 

        
        for i in range(10):
            if i == 3:
                continue
            if i == 7:
                break
            print(i)

هذا الكود يوضح استخدام continue و break معًا داخل حلقة for

الحلقة تبدأ من 0 إلى 9

عندما i == 3 يتم تخطي الطباعة بسبب continue

وعندما i == 7 تتوقف الحلقة تمامًا بسبب break

النتيجة: طباعة 0، 1، 2، 4، 5، 6

القوائم (Lists)

هي مجموعة من القيم نخزنها في متغير واحد

        
        numbers = [1, 2, 3, 4, 5]
        print(numbers)

هذا الكود يوضح القوائم List في Python

numbers هي قائمة تحتوي على عدة أرقام

القائمة تُكتب بين أقواس مربعة [ ]

يمكن تخزين أكثر من قيمة في متغير واحد

print(numbers) يقوم بطباعة جميع العناصر داخل القائمة

الوصول إلى العناصر

الفهرسة تبدأ من 0

        
         fruits = ["apple", "banana", "orange"]

         print(fruits[0])  # apple
         print(fruits[1])  # banana

هذا الكود يوضح الوصول إلى عناصر القائمة List في Python

القائمة fruits تحتوي على عدة عناصر نصية

نستخدم الفهرس [index] للوصول إلى العنصر

الفهرس يبدأ من 0، لذلك fruits[0] هو "apple"

وfruits[1] هو "banana"

تعديل عنصر

        
         fruits = ["apple", "banana", "orange"]
 
         fruits[1] = "grape"
         print(fruits)

هذا الكود يوضح تعديل عناصر القائمة في Python

القائمة fruits تحتوي على عناصر نصية

fruits[1] = grape يقوم بتغيير العنصر الثاني

الفهرس 1 يشير إلى "banana" ويتم استبداله بـ "grape"

النتيجة: ["apple", "grape", "orange"].

إضافة عناصر

        
         fruits = ["apple", "banana"]

         fruits.append("orange")  # إضافة عنصر
         print(fruits)

هذا الكود يوضح إضافة عنصر إلى القائمة في Python

القائمة fruits تحتوي على عنصرين في البداية

append("orange") تضيف عنصرًا جديدًا في نهاية القائمة

لا يتم استبدال العناصر بل يتم إضافة عنصر جديد

النتيجة: ["apple", "banana", "orange"]

حذف عناصر

        
         fruits = ["apple", "banana", "orange"]

         fruits.remove("banana")
         print(fruits)

هذا الكود يوضح حذف عنصر من القائمة في Python

القائمة fruits تحتوي على عدة عناصر

remove("banana") تقوم بحذف العنصر "banana" من القائمة

يتم الحذف بناءً على القيمة وليس الفهرس

النتيجة: ["apple", "orange"]

         for fruit in fruits:

القائمة fruits تحتوي على عدة عناصر

الحلقة for تمر على كل عنصر واحدًا تلو الآخر

النطاق (Scope - Global / Local)

ما هو Scope

هو المكان الذي يمكن الوصول فيه إلى المتغير

المتغير المحلي Local

يُعرّف داخل دالة لا يمكن استخدامه خارجها

        
          def my_func():
              x = 10  # متغير محلي
              print(x)

          my_func()

          # print(x)  خطأ

هذا الكود يوضح المتغيرات المحلية Local Variables في Python

المتغير x = 10 تم تعريفه داخل الدالة ()my_func

داخل الدالة يمكن استخدام x وطباعته

عند استدعاء ()my_func يتم طباعة 10

خارج الدالة، محاولة استخدام x تسبب خطأ لأنه متغير محلي لا يمكن الوصول له خارج الدالة

المتغير العام Global

يُعرّف خارج الدوال يمكن استخدامه في كل مكان

        
        x = 5  # متغير عام

        def my_func():
            print(x)

        my_func()

هذا الكود يوضح المتغيرات العامة Global Variables في Python

المتغير x = 5 تم تعريفه خارج أي دالة، لذا هو متغير عام

الدالة ()my_func يمكنها الوصول إلى المتغير العام مباشرة

عند استدعاء ()my_func يتم طباعة 5

المتغير العام يمكن استخدامه داخل أي دالة إلا إذا تم تغييره محليًا بنفس الاسم

الفرق بين المتغير المحلي والمتغير العام Local vs Global Variables

         greet()

هذا الكود يوضح الفرق بين المتغير المحلي والمتغير العام Local vs Global Variables في Python

name = "Mubarak" هو متغير عام (Global)

داخل الدالة greet() تم تعريف name = "Ali"، وهو متغير محلي (Local) يظل موجودًا فقط داخل الدالة

print("Inside:", name) داخل الدالة يطبع "Ali"

print("Outside:", name) خارج الدالة يطبع "Mubarak" لأن المتغير العام لم يتغير

هذا الكود يمثل أداة برمجية بسيطة تُعرف بـ ماسح الشبكة Network Scanner أو Ping Sweeper، وهدفها الأساسي هو اكتشاف الأجهزة المتصلة والنشطة داخل شبكة محلية معينة. تعتمد فكرة السكربت على استغلال بروتوكول ICMP عبر أمر ping الشهير؛ حيث يقوم البرنامج بإنشاء حلقة تكرارية تمر على جميع العناوين المحتملة في نطاق الشبكة من 1 إلى 254 ويرسل لكل عنوان إشارة برمجية واحدة. من الناحية الوظيفية، يقوم الكود بدمج الجزء الثابت من عنوان الشبكة مع الرقم المتغير لكل جهاز ليحصل على عنوان IP كامل، ثم ينفذ أمر النظام لإرسال حزمة بيانات وانتظار الرد خلال ثانية واحدة كحد أقصى. إذا استجاب الجهاز المستهدف، فإن نظام التشغيل يعيد قيمة "صفر"، وهو ما يفسره السكربت بأن الجهاز نشط LIVE ويقوم بطباعة عنوانه للمستخدم. تكمن أهمية هذا الأسلوب في قدرته على رسم خريطة أولية للأجهزة المتصلة بالشبكة، مما يساعد مديري الأنظمة في مراقبة الاتصالات أو يساعد مختبري الاختراق في تحديد الأهداف المحتملة قبل البدء في عمليات فحص أكثر تعقيداً للمنافذ والثغرات.

رابط الاسكربت
SQL Injection Scanner

يعتبر هذا الكود نموذجاً مبسطاً لأداة فحص آلية تهدف إلى اكتشاف ثغرات حقن الاستعلامات SQL Injection في مواقع الويب. تعتمد الفكرة الأساسية للسكربت على مبدأ اختبار الاستجابة للخطأ حيث يقوم البرنامج بحقن رموز برمجية خبيثة تُعرف بالـ Payloads داخل رابط الموقع المستهدف، مثل رمز الفاصلة العليا ' أو عبارات منطقية دائمة التحقق مثل ' OR '1'='1. الهدف من هذه المدخلات هو خداع قاعدة بيانات الموقع ومحاولة كسر منطق الاستعلام البرمجي الذي تعمل به. من الناحية العملية، يستخدم الكود مكتبة requests لإرسال هذه الرموز ومراقبة رد فعل السيرفر فبدلاً من الحصول على صفحة ويب طبيعية، يبحث السكربت في نص الاستجابة عن كلمات دلالية معينة مثل "SQL syntax" أو "mysql_fetch". وجود هذه الكلمات في رد الموقع يعني أن قاعدة البيانات قد ارتبكت وأظهرت خطأً تقنياً للمستخدم، وهو مؤشر قوي على أن الموقع لا يقوم بتصفية المدخلات بشكل آمن. وبالرغم من أن هذا السكربت يكتشف الأنواع الظاهرة فقط من الثغرات Error-based SQLi إلا أنه يوضح بجلاء كيف يمكن للمهاجمين استغلال ضعف البرمجة للوصول إلى معلومات حساسة أو تجاوز أنظمة تسجيل الدخول.

رابط الاسكربت
XSS Scanner

يُصنف هذا الكود كأداة فحص آلية بسيطة لاكتشاف ثغرات حقن النصوص البرمجية Cross-Site Scripting - XSS وهي واحدة من أكثر الثغرات شيوعاً في تطبيقات الويب. تتمحور فكرة السكربت حول اختبار مدى أمان الموقع عند التعامل مع مدخلات المستخدم حيث يقوم البرنامج بحقن أكواد جافا سكريبت JavaScript صغيرة داخل رابط الموقع مثل أمر وينتظر ليرى كيف سيتعامل المتصفح معها. يعتمد السكربت في عمله على مبدأ الانعكاس Reflection فهو يرسل الكود الخبيث Payload ضمن طلب HTTP عبر مكتبة requests ثم يفحص محتوى الصفحة الناتجة. إذا وجد البرنامج أن الكود الذي أرسله قد ظهر كما هو في نص استجابة الموقع فهذا يعني أن الموقع يرتكب خطأً فادحاً بإعادة عرض مدخلات المستخدم دون تنقيتها أو تشفيرها. هذا النوع من الثغرات يسمح للمهاجمين بتنفيذ أوامر برمجية داخل متصفحات الزوار الآخرين مما قد يؤدي لسرقة ملفات تعريف الارتباط Cookies أو السيطرة على جلسات المستخدمين وهو ما يحاول هذا الكود كشفه بشكل أولي ومبسط.

رابط الاسكربت
Automation Scripts

هذا الكود يمثل سكربت بسيطاً لـ أتمتة أوامر النظام System Commands Automation ويهدف إلى تنفيذ سلسلة من الأوامر التشغيلية دفعة واحدة بدلاً من إدخالها يدوياً في واجهة السطر البرمجي Terminal. تعتمد فكرة الكود على الربط بين لغة بايثون ونظام التشغيل الأساسي مثل Linux أو macOS حيث يتم تخزين مجموعة من الأوامر الاستكشافية في قائمة نصية ثم يقوم البرنامج بالمرور عليها وتنفيذها تتابعاً. من الناحية الوظيفية يستخدم الكود مكتبة os وتحديداً دالة os.system التي تعمل كجسر لنقل الأوامر من بيئة بايثون إلى صدفة النظام Shell. الأوامر المكتوبة في الكود مثل whoami لمعرفة المستخدم الحالي و pwd لتحديد المسار و ls -la لعرض الملفات هي أوامر قياسية تستخدم لجمع المعلومات الأساسية عن بيئة العمل. تكمن أهمية هذا الأسلوب في تسهيل مهام إدارة الأنظمة واختصار الوقت في فحص إعدادات الجهاز أو استخراج تقارير سريعة عن حالة النظام وهو يمثل الخطوة الأولى في بناء أدوات التحكم عن بُعد والبرمجيات التي تتفاعل مباشرة مع موارد الحاسوب.

رابط الاسكربت
Scan + Save Results

يعبر هذا الكود عن أداة بسيطة لـ نسخ محتوى صفحات الويب Web Page Downloader وهي تقنية أساسية تُستخدم للأرشفة أو لتحليل بنية المواقع محلياً. تعتمد فكرة السكربت على محاكاة سلوك متصفح الإنترنت في طلب البيانات حيث يقوم البرنامج بإرسال طلب استدعاء للموقع عبر الرابط الذي يدخله المستخدم ثم يستقبل الشيفرة المصدرية الكاملة HTML التي يتكون منها الموقع. من الناحية التقنية يستخدم الكود مكتبة requests لإجراء عملية الاتصال والحصول على نص الاستجابة ثم ينتقل إلى مرحلة التخزين الفيزيائي باستخدام دوال التعامل مع الملفات في بايثون. يقوم السكربت بإنشاء ملف جديد بصيغة html على جهاز المستخدم وكتابة كافة البيانات المستلمة بداخله. تكمن أهمية هذه العملية في كونها الخطوة الأولى لمهندسي البرمجيات الراغبين في بناء أدوات الزحف الشبكي Web Crawling أو الباحثين الأمنيين الذين يحتاجون لتحليل الكود المصدري للمواقع بعيداً عن المتصفح لاكتشاف الروابط المخفية أو الثغرات البرمجية في بنية الصفحة.

رابط الاسكربت
Brute Force

هذا الكود يمثل نموذجاً برمجياً لهجوم يُعرف بـ التخمين أو القوة الغاشمة (Brute Force Attack)، وتحديداً على صفحات تسجيل الدخول في المواقع الإلكترونية. تعتمد فكرة السكربت على محاولة اقتحام حساب معين (مثل حساب "admin") من خلال تجربة قائمة من كلمات المرور الشائعة أو المحتملة بشكل آلي وسريع، بدلاً من إدخالها يدوياً، وذلك أملاً في مطابقة الكلمة الصحيحة وتجاوز نظام الحماية. من الناحية التقنية، يستخدم الكود مكتبة requests لإرسال طلبات من نوع POST إلى رابط تسجيل الدخول، وهي الطلبات التي تحمل عادةً بيانات اسم المستخدم وكلمة المرور. في كل دورة داخل الحلقة التكرارية، يرسل البرنامج كلمة مرور مختلفة من القائمة المخزنة لديه، ثم يقوم بفحص نص الاستجابة القادم من السيرفر؛ فإذا وجد كلمة ترحيبية مثل "Welcome"، يستنتج السكربت أن عملية تسجيل الدخول قد نجحت وأن الكلمة المستخدمة هي الكلمة الصحيحة. يُبرز هذا الكود أهمية استخدام كلمات مرور معقدة وتفعيل خاصية "قفل الحساب" بعد عدد معين من المحاولات الفاشلة، حيث أن الأدوات البرمجية كهذه قادرة على تجربة آلاف الاحتمالات في وقت قياسي إذا لم توجد قيود أمنية برمجية تمنعها.

رابط الاسكربت
Recon Script

يعبر هذا الكود عن أداة برمجية بسيطة مخصصة لـ استعلامات سجلات الأسماء (DNS Lookup)، وهدفها الأساسي هو الربط بين أسماء النطاقات (مثل https://www.google.com/search?q=google.com) وعناوين البروتوكول الرقمية (IP addresses). تعتمد فكرة السكربت على وظيفة "المترجم" التي يقوم بها نظام أسماء النطاقات في الإنترنت، حيث يسهل على البشر استخدام الأسماء المكتوبة بينما يسهل على الآلات استخدام الأرقام. من الناحية التقنية، يستخدم الكود مكتبة socket لتنفيذ عمليتين متكاملتين؛ الأولى هي "التحويل الأمامي" عبر دالة gethostbyname للحصول على عنوان الـ IP الخاص بالنطاق المدخل. أما الثانية فهي "التحويل العكسي" (Reverse DNS) عبر دالة gethostbyaddr التي تحاول معرفة اسم المضيف المرتبط بهذا العنوان الرقمي. تكمن أهمية هذا السكربت في كونه أداة استكشافية أولية يستخدمها مهندسو الشبكات للتحقق من صحة ربط النطاقات بسيرفراتها، أو من قبل الباحثين الأمنيين لتحديد المواقع الجغرافية للسيرفرات ومعرفة مزودي الخدمة التابعين لها.

رابط الاسكربت
Exploitation Scripts

يعتبر هذا الكود نموذجاً توضيحياً لهجوم خطير يُعرف بـ حقن الأوامر (Command Injection)، وتتمحور فكرته حول استغلال ضعف في معالجة المدخلات داخل تطبيقات الويب التي تتفاعل مباشرة مع نظام التشغيل. تعتمد الفكرة على خداع الموقع لتنفيذ أوامر إضافية لم يكن من المفترض تنفيذها، وذلك عن طريق دمجها مع المدخلات المتوقعة باستخدام رموز خاصة مثل الفاصلة المنقوطة (;) التي تفصل بين الأوامر في أنظمة مثل Linux. من الناحية التقنية، يفترض السكربت وجود رابط يقوم بوظيفة شرعية (مثل عمل ping لعنوان IP)، ولكنه لا يقوم بتصفية المدخلات بشكل كافٍ. يقوم الكود بإرسال "حموله" (Payload) تحتوي على عنوان محلي متبوعاً بأمر نظام مثل whoami؛ فإذا كان الموقع مصاباً بهذه الثغرة، سيقوم السيرفر بتنفيذ الأمر الأول ثم يتبعه بتنفيذ الأمر المحقون ويعيد النتيجة للمهاجم. تكمن خطورة هذا النوع من الثغرات في أنها قد تمنح المهاجم سيطرة كاملة على السيرفر، حيث تتيح له استعراض الملفات، تعديل البيانات، أو حتى إغلاق النظام بالكامل، مما يجعل من تنقية المدخلات (Input Validation) حائط الصد الأول والأساسي للمبرمجين.

رابط الاسكربت
SQL Injection Exploit

يُصنف هذا الكود كأداة لاختبار نوع متقدم من هجمات حقن الاستعلامات يُعرف بـ حقن SQL القائم على الاتحاد (UNION-Based SQL Injection). تكمن الفكرة الجوهرية هنا في استغلال أمر UNION في لغة قواعد البيانات، والذي يُستخدم لدمج نتائج استعلامين مختلفين في جدول واحد؛ حيث يحاول المهاجم إجبار قاعدة البيانات على إظهار معلومات حساسة لا تظهر عادةً في الصفحة، مثل اسم قاعدة البيانات أو بيانات المستخدمين، عبر دمجها مع النتائج الطبيعية التي يعرضها الموقع. من الناحية التقنية، يقوم السكربت بحقن "حمولة" (Payload) تبدأ بكسر الاستعلام الأصلي باستخدام الفاصلة العليا ' ثم إضافة أمر UNION SELECT. وفي هذا المثال تحديداً، يحاول الكود استخراج اسم قاعدة البيانات الحالية باستخدام الدالة database()، مع استخدام null لضبط عدد الأعمدة حتى يتطابق مع الاستعلام الأصلي، وينتهي بـ -- لإهمال بقية الكود البرمجي الأصلي للسيرفر. تكمن خطورة هذا الهجوم في قدرته على استخراج كميات ضخمة من البيانات من جداول مختلفة داخل القاعدة، مما يجعله وسيلة فعالة جداً في عمليات تسريب البيانات (Data Exfiltration) إذا لم يتم تحصين الموقع باستخدام "الاستعلامات المُجهزة" (Parameterized Queries).

Brute Force Login

يعتبر هذا الكود نموذجاً برمجياً لهجوم يُعرف بـ التخمين أو القوة الغاشمة (Brute Force Attack)، وتحديداً على صفحات تسجيل الدخول في المواقع الإلكترونية. تعتمد فكرة السكربت على محاولة اقتحام حساب معين (مثل حساب "admin") من خلال تجربة قائمة من كلمات المرور الشائعة أو المحتملة بشكل آلي وسريع، بدلاً من إدخالها يدوياً، وذلك أملاً في مطابقة الكلمة الصحيحة وتجاوز نظام الحماية. من الناحية التقنية، يستخدم الكود مكتبة requests لإرسال طلبات من نوع POST إلى رابط تسجيل الدخول، وهي الطلبات التي تحمل عادةً بيانات اسم المستخدم وكلمة المرور. في كل دورة داخل الحلقة التكرارية، يرسل البرنامج كلمة مرور مختلفة من القائمة المخزنة لديه، ثم يقوم بفحص نص الاستجابة القادم من السيرفر؛ فإذا وجد كلمة ترحيبية مثل "Welcome"، يستنتج السكربت أن عملية تسجيل الدخول قد نجحت وأن الكلمة المستخدمة هي الكلمة الصحيحة. يُبرز هذا الكود أهمية استخدام كلمات مرور معقدة وتفعيل خاصية "قفل الحساب" بعد عدد معين من المحاولات الفاشلة، حيث أن الأدوات البرمجية كهذه قادرة على تجربة آلاف الاحتمالات في وقت قياسي إذا لم توجد قيود أمنية برمجية تمنعها.

رابط الاسكربت
Reverse Shell

يمثل الكود الثاني نموذجاً برمجياً لما يُعرف بـ القشرة العكسية (Reverse Shell)، وهي تقنية تسمح للمهاجم بالتحكم في جهاز الضحية عن بُعد عبر الالتفاف على جدران الحماية. تكمن الفكرة الجوهرية هنا في قلب موازين الاتصال التقليدي؛ فبدلاً من أن يحاول المهاجم الاتصال بجهاز الضحية (وهو ما تمنعه معظم برامج الحماية عادةً)، يقوم جهاز الضحية نفسه بإنشاء الاتصال والخروج نحو جهاز المهاجم "طواعية". من الناحية الوظيفية، يستخدم السكربت مكتبة socket لفتح اتصال مباشر بعنوان الـ IP الخاص بالمهاجم، ثم يدخل في حلقة تكرارية لا تنتهي ينتظر فيها استقبال "أوامر" نصية. بمجرد وصول الأمر، يستخدم الكود مكتبة subprocess لتنفيذ هذا الأمر داخل نظام تشغيل الضحية وكأنه مكتوب محلياً، ثم يرسل نتيجة التنفيذ (مثل محتويات الملفات أو معلومات النظام) مرة أخرى إلى المهاجم. تُعد القشرة العكسية من أخطر الوسائل المستخدمة في الاختراقات لأنها تمنح المهاجم واجهة تحكم كاملة (Terminal) وتجعل من الصعب اكتشاف الاتصال كونه يظهر في الشبكة كحركة مرور عادية خارجة من الجهاز.

رابط الاسكربت
File Upload Exploit

يعبر هذا الكود الأخير عن هجوم سيبراني شديد الخطورة يُعرف باسم رفع الملفات الضار (Malicious File Upload)، وتحديداً رفع ما يسمى بـ Web Shell. تكمن الفكرة الجوهرية هنا في استغلال ميزة برمجية تسمح للمستخدمين برفع ملفات إلى الموقع، واستغلالها لإرسال ملف يحتوي على شيفرة برمجية خبيثة بلغة مثل PHP بدلاً من الصور أو الوثائق العادية. من الناحية التقنية، يستخدم الكود مكتبة requests لإرسال طلب من نوع POST إلى نموذج الرفع في الموقع المستهدف. يقوم السكربت بإنشاء ملف وهمي يحمل الاسم shell.php ويضع بداخله أمراً برمجياً قوياً (system($_GET['cmd']))؛ هذا الأمر يسمح للمهاجم بمجرد رفع الملف بنجاح، بتنفيذ أي أمر يريده على سيرفر الموقع مباشرة عبر المتصفح. تكمن خطورة هذه الثغرة في أنها تمنح المهاجم "موطئ قدم" دائم داخل السيرفر، مما قد يؤدي إلى سرقة كامل قواعد البيانات أو حتى مسح ملفات الموقع بالكامل، وهي تبرز أهمية قيام المطورين بفحص نوع الملفات المرفوعة بدقة وتغيير أسمائها وتخزينها في مسارات غير قابلة للتنفيذ.

يمثل الكود الأخير أداة تقنية متقدمة تُعرف بـ شامّ الحزم (Packet Sniffer)، وهي أداة تسمح "بالتسمع" على حركة مرور البيانات داخل الشبكة. تكمن الفكرة الجوهرية للسكربت في القدرة على التقاط وفحص "طرود البيانات" (Packets) التي تنتقل بين الأجهزة عبر الشبكة، مما يتيح للمحلل رؤية ما يحدث "خلف الكواليس" في الاتصالات الرقمية.

رابط الاسكربت
Packet Sniffer

يُعد هذا الكود نموذجاً برمجياً لوظيفة تقنية هامة في إدارة الخوادم وأمن الشبكات تُعرف بـ فحص المنافذ المفتوحة (Port Scanner). تكمن الفكرة الجوهرية للسكربت في محاولة "طرق الأبواب الرقمية" لجهاز معين (IP Address) لمعرفة أي من هذه الأبواب (المنافذ) مفتوح ومستعد لاستقبال البيانات، وأيها مغلق أو محمي.

رابط الاسكربت
HTTP Requests

هذا السكريبت هو أداة "تنصت" برمجية تستخدم مكتبة Scapy لالتقاط حزم البيانات عبر المنفذ 80، ثم استخراج وتحليل روابط HTTP التي يزورها المستخدم وعرضها مباشرة.

Extracting sensitive data

هذا السكريبت هو محلل بيانات (Data Sniffer) يبحث داخل حزم الشبكة عن كلمات مفتاحية محددة (مثل كلمة المرور) لاستخراج بيانات تسجيل الدخول التي يتم إرسالها بشكل غير مشفر.

DNS Sniffer

هذا السكريبت هو مراقب لطلبات DNS (DNS Sniffer) يقوم برصد وتحليل أسماء المواقع التي يحاول المستخدمون على الشبكة الاتصال بها عبر مراقبة المنفذ 53.

API Interaction Scripts

هذا السكريبت يقوم بإرسال طلب استعلام (GET Request) إلى واجهة برمجة تطبيقات GitHub لجلب بيانات مستخدم معين بصيغة JSON وعرضها.

POST Request

JSON Parsing

يقوم هذا السكريبت بجلب بيانات تقنية من API الخاص بـ GitHub، ثم يقوم بـ "تصفيتها" لاستخراج وعرض اسم المستخدم وعدد متابعيه بشكل منظم.

رابط الاسكربت
Bash

هي اختصار لـ "Bourne Again SHell"، وهي لغة أوامر نصية Scripting Language ومفسّر سطر أوامر Command Line Interpreter قوية تُستخدم بشكل أساسي في أنظمة التشغيل القائمة على Unix و Linux لإدارة النظام، أتمتة المهام، والتفاعل المباشر مع النواة

بمجرد فتح التيرمنال، أنت غالباً تستخدم Bash بالفعل. للتأكد من ذلك، يمكنك كتابة الأمر التالي:

echo $SHELL

إذا كانت النتيجة /bin/bash أو /usr/bin/bash فالباش يعمل حالياً.

إذا كان التيرمنال الخاص بك يستخدم نوعاً آخر (مثل Zsh أو Fish)، يمكنك ببساطة كتابة:

bash

سيتحول المحث (Prompt) فوراً للعمل ببيئة Bash. وللخروج منها والعودة للصدّفة السابقة، اكتب exit.

        
        #!/bin/bash
        echo "Hello, World!"
        
      
#!/bin/bash:

يُسمى "Shebang"، وهو يُخبر النظام بضرورة استخدام مفسر أوامر Bash لتنفيذ هذا الملف

echo: هو أمر برمجي يُستخدم لطباعة أو عرض النصوص والبيانات على الشاشة المخرجات

"Hello, World!": هو السلسلة النصية المراد طباعتها، وتوضع بين علامتي تنصيص للحفاظ على تنسيقها

قوم هذا السكربت البسيط عند تشغيله بإظهار عبارة الترحيب الشهيرة للمستخدم في واجهة الأوامر

يتطلب السكربت عادةً إعطاء ملفه صلاحية "التنفيذ" عبر الأمر chmod +x ليعمل بشكل صحيح

المتغيرات (Variables)

بدون مسافات حول = تستخدم $ لعرض القيمة

        
         #!/bin/bash

         name="Mubarak"
         age=22

         echo "My name is $name"
         echo "I am $age years old"
        
      
#!/bin/bash:

هو السطر التعريفي (Shebang) الذي يُحدد للمواد البرمجية في نظام التشغيل أن هذا الملف يجب تنفيذه باستخدام مفسر Bash

السطران name="Mubarak" و age=22 يقومان بتخزين القيم النصية والعددية داخل متغيرات لاستدعائها لاحقاً، مع ملاحظة عدم وجود مسافات حول علامة =

عند كتابة $name أو $age داخل نص، يفهم النظام أنك تريد استبدال اسم المتغير بالقيمة الفعلية المخزنة داخله (مثل Mubarak)

echo: يُستخدم لطباعة النصوص الناتجة على الشاشة، حيث يقوم في هذا السكربت بدمج النصوص الثابتة مع قيم المتغيرات

عند تشغيل هذا السكربت، ستظهر للمستخدم جملتان واضحتان: "My name is Mubarak" و "I am 22 years old"

إدخال من المستخدم (User input) 
        
         #!/bin/bash

         echo "Enter your name:"
         read name

         echo "Welcome $name!"
        
      
#!/bin/bash:

يحدد مسار مفسر الأوامر (Bash) الذي سيقوم بقراءة وتنفيذ التعليمات البرمجية الموجودة في الملف

echo الأول: يقوم بطباعة نص توضيحي على الشاشة يطلب من المستخدم إدخال اسمه (Enter your name)

read name: هو أمر حيوي يقوم بوقف تنفيذ السكربت مؤقتاً لانتظار إدخال من المستخدم عبر لوحة المفاتيح، ثم يخزن ما يكتبه المستخدم في متغير يسمى name

يستخدم الأمر echo مرة أخرى لطباعة رسالة ترحيبية، حيث يتم استدعاء القيمة التي أدخلها المستخدم برمجياً باستخدام الرمز $

يمثل هذا الكود أبسط أشكال "إدخال البيانات ومعالجتها ثم إخراجها" (Input-Process-Output) في برمجة السكربتات

الشروط (if)

الشروط (if)

-gt أكبر

-lt أصغر

-eq يساوي

        
         #!/bin/bash

         echo "Enter a number:"
         read num

         if [ $num -gt 10 ]; then
         echo "Number is greater than 10"
         else
         echo "Number is 10 or less"
         fi

يبدأ السكربت بطلب رقم من المستخدم عبر echo ويقوم بتخزينه في المتغير num باستخدام الأمر read

if: تبدأ الجملة الشرطية بـ ;if [ $num -gt 10 ] حيث أن الاختصار -gt يعني "أكبر من" (Greater Than)، وهي تختبر القيمة المدخلة

then: إذا كانت القيمة الموجودة في $num أكبر فعلياً من 10، يتم تنفيذ الأمر البرمجي الذي يلي كلمة then مباشرة

else: في حال كان الشرط خاطئاً (أي أن الرقم 10 أو أقل)، يتم الانتقال تلقائياً لتنفيذ الأوامر الموجودة تحت قسم else

fi: تنتهي جملة if دائماً بكلمة fi (وهي كلمة if مكتوبة بشكل معكوس)، لتعريف النظام بأن منطق المقارنة قد انتهى هنا

الحلقات (Loops)

for loop

        
         #!/bin/bash

         for i in 1 2 3 4 5
         do
         echo "Number: $i"
         done

for: تبدأ الحلقة بـ for i in 1 2 3 4 5 حيث يتم إنشاء متغير مؤقت اسمه i يأخذ قيمة جديدة من القائمة الموجودة (1 إلى 5) في كل دورة

do: كلمة do تعني "افعل"، وهي تشير إلى بداية الأوامر التي سيتم تكرارها طالما لم تنتهِ قائمة الأرقام المحددة

يقوم الأمر "echo "Number: $i بطباعة الكلمة متبوعة بالقيمة الحالية للمتغير i؛ لذا ستشاهد الأرقام تُطبع واحداً تلو الآخر في كل سطر

done: تُستخدم كلمة done لإعلام مفسر Bash بأن مجموعة الأوامر المطلوب تكرارها قد انتهت، ليعود البرنامج إلى البداية لاختيار الرقم التالي

تُغني هذه الطريقة عن كتابة أمر الطباعة يدوياً خمس مرات، مما يوفر الوقت ويقلل الأخطاء البرمجية عند التعامل مع بيانات كبيرة

while loop

        
         #!/bin/bash

         count=1

         while [ $count -le 5 ]
         do
         echo "Count: $count"
         ((count++))
         done

يبدأ الكود بـ count=1 لضبط نقطة انطلاق العداد من الرقم واحد.

جملة while [ $count -le 5 ] تعني "استمر طالما أن العداد أقل من أو يساوي (Less than or Equal) الرقم 5".

do: تُحدد هذه الكلمة بداية الكتلة البرمجية التي سيتم تكرار تنفيذها طالما ظل الشرط محققاً.

((count++)): هذا السطر في غاية الأهمية، فهو يقوم بزيادة قيمة العداد بمقدار 1 في كل دورة؛ وبدونه سيدخل السكربت في "حلقة لانهائية" لأن الشرط سيظل دائماً صحيحاً.

done: تُنهي الحلقة وتُعيد البرنامج إلى سطر الشرط للتأكد مما إذا كان يجب التكرار مرة أخرى أم الخروج من الحلقة.

التعامل مع الملفات (Handling files)

إنشاء وكتابة:

        
          #!/bin/bash

          echo "Hello Bash" > file.txt
        
      
echo "Hello Bash":

يقوم المحرك هنا بتجهيز النص "Hello Bash" للطباعة، لكنه لن يظهر على الشاشة هذه المرة

هذا الرمز يُخبر النظام: "لا تطبع النص على الشاشة، بل أرسله إلى مكان آخر"، وفي هذه الحالة هو ملف نصي

file.txt: إذا لم يكن الملف موجوداً، سيقوم السكربت بإنشائه تلقائياً، أما إذا كان موجوداً فسيقوم بمسح محتواه القديم وكتابة النص الجديد مكانه

يُسمى هذا الإجراء "Standard Output Redirection"، وهو وسيلة أساسية لحفظ نتائج الأوامر أو سجلات العمل (Logs) للرجوع إليها لاحقاً

إذا استخدمت > فستستبدل المحتوى، بينما إذا استخدمت >> فستضيف النص الجديد إلى نهاية الملف دون مسح القديم

قراءة:

        
         #!/bin/bash

         cat file.txt

cat: هو اختصار لكلمة (Concatenate)، ورغم أن اسمه يوحي بدمج الملفات، إلا أن استخدامه الأكثر شيوعاً هو عرض محتويات الملفات النصية

file.txt: يمثل اسم الملف الذي يرغب السكربت في الوصول إليه. يجب أن يكون الملف موجوداً في نفس المجلد الذي يُنفذ فيه السكربت ليعمل بنجاح

يقوم الأمر بفتح الملف وقراءة البيانات المخزنة داخله (مثل نص "Hello Bash" الذي كتبناه سابقاً) ثم دفعها مباشرة إلى شاشة الطرفية (Terminal)

بمجرد تشغيل السكربت، سيظهر لك محتوى الملف كنص عادي أمامك، مما يجعله أداة سريعة للتحقق من البيانات دون الحاجة لفتح محرر نصوص

يمكنك استخدام cat مع عدة ملفات في آن واحد (مثلاً cat file1.txt file2.txt) ليقوم بعرضها جميعاً خلف بعضها البعض كتسلسل واحد

قراءة سطر بسطر:

        
         #!/bin/bash

         while read line
         do
         echo "Line: $line"
         done < file.txt

حلقة while read line: تقوم هذه الحلقة بقراءة الملف سطرًا واحدًا في كل دورة، وتخزن محتوى السطر الحالي داخل متغير يسمى line

بداية التنفيذ do: تبدأ تنفيذ الأوامر على السطر الذي تم التقاطه للتو من الملف

معالجة البيانات: الأمر echo "Line: $line" يعرض النص الموجود في السطر مسبوقاً بكلمة ":Line"، مما يساعدك على تمييز كل سطر بشكل منفصل

إعادة توجيه الإدخال < file.txt: العلامة < تخبر حلقة while بأن مصدر البيانات ليس لوحة المفاتيح، بل هو محتوى الملف file.txt

هذه الطريقة أفضل من استخدام cat داخل حلقة، لأنها تستهلك ذاكرة أقل وتتعامل مع الملفات الكبيرة بكفاءة عالية دون تحميل الملف بالكامل في الذاكرة دفعة واحدة

التعامل مع المستخدمين والملفات (Handling users and files) 
        
        #!/bin/bash

        echo "Checking if file exists..."

        if [ -f file.txt ]; then
        echo "File exists"
        else
        echo "File not found"
        fi

يبدأ السكربت بطباعة نص توضيحي "Checking if file exists..." ليُعلم المستخدم بالعملية التي تجري الآن.

[ -f file.txt ]: المعامل -f (اختصار لـ File) هو اختبار منطقي يتحقق من شيئين: أن الملف موجود، وأنه ملف عادي (وليس مجلداً).

إذا وجد السكربت الملف file.txt في المسار المحدد، سيقوم بتنفيذ الأمر الأول وطباعة "File exists".

else: إذا كان الملف مفقوداً أو تم حذفه، سينتقل السكربت تلقائياً لطباعة رسالة التنبيه "File not found".

يُستخدم هذا النمط بكثرة في "الأتمتة"، حيث لا يمكنك قراءة ملف أو تعديله قبل التأكد من وجوده فعلياً لمنع تعطل السكربت.

Advanced Network Scanner

يهدف هذا السكربت البرمجي المكتوب بلغة Bash إلى أتمتة عملية فحص الشبكات باستخدام أداة Nmap الشهيرة، حيث يبدأ باستقبال عنوان الهدف (IP أو Domain) كمتغير يتم تمريره عند تشغيل الملف ليقوم بعد ذلك بسلسلة من الفحوصات المتدرجة التي تبدأ بالفحص السريع للمنافذ الأكثر شيوعاً عبر الخيار -F. يتوسع السكربت لاحقاً ليشمل تحليل الخدمات النشطة وإصداراتها باستخدام -sV ومحاولة التعرف على نظام التشغيل الخاص بالهدف عبر الخيار -O ليعطي صورة متكاملة عن البيئة التقنية المستهدفة. وفي المرحلة الختامية ينفذ السكربت فحصاً شاملاً وعدوانياً بواسطة الخيار -A الذي يجمع بين اكتشاف الأنظمة والخدمات وتشغيل السكربتات المساعدة، مع توجيه وحفظ كافة النتائج النهائية في ملف نصي خارجي يسمى scan_result.txt لضمان توثيق البيانات وسهولة مراجعتها لاحقاً.

Backup

يعمل هذا السكربت كأداة ذكية للأتمتة تهدف إلى إنشاء نسخ احتياطية مؤرخة للملفات لضمان عدم ضياع البيانات وتسهيل تتبع التغييرات عبر الزمن. تبدأ العملية بتعريف متغير المصدر src الذي يحتوي على الملف المراد نسخه، يليه الجزء الأكثر إبداعاً وهو تكوين اسم ملف النسخة الاحتياطية باستخدام أمر النظام $(date +%Y%m%d)، والذي يقوم بدمج التاريخ الحالي (السنة والشهر واليوم) تلقائياً داخل اسم الملف ليصبح فريداً لكل يوم. بعد ذلك، يتم تنفيذ أمر النسخ cp الذي ينقل المحتويات من الملف الأصلي إلى الملف الجديد المسمى بالتاريخ، ثم يختتم السكربت عمله بطباعة رسالة تأكيدية للمستخدم توضح الاسم الدقيق للنسخة التي تم إنشاؤها بنجاح.

رابط الاسكربت
Full Backup Script

يُقدم هذا السكربت حلاً احترافياً لأرشفة وضغط البيانات، حيث ينتقل من مجرد نسخ الملفات إلى تقليص حجمها وتجميعها في ملف واحد محمي زمنياً. يبدأ الكود بتحديد مسار البيانات المصدر SOURCE والوجهة النهائية للنسخ DEST داخل نظام الملفات، مع استخدام متغير التاريخ DATE لضمان الحصول على طابع زمني دقيق لكل عملية أرشفة. تكمن القوة الحقيقية في الأمر tar -czf؛ حيث يقوم الحرف c بإنشاء الأرشيف، وz بضغطه باستخدام خوارزمية gzip لتقليل المساحة التخزينية، بينما يحدد f اسم الملف الناتج. بهذه الطريقة، يتم تحويل مجلد كامل من البيانات إلى ملف واحد مضغوط بصيغة .tar.gz يحمل تاريخ اليوم، مما يسهل عملية النقل والتخزين طويل الأمد مع الحفاظ على هيكلية المجلدات الأصلية.

رابط الاسكربت
Log Cleanup Script

صُمم هذا السكربت لأداء مهمة إدارة وصيانة مساحة التخزين عبر تنظيف سجلات النظام (Logs) القديمة بشكل آلي لمنع امتلاء القرص الصلب. يعتمد الكود بشكل أساسي على القوة البرمجية للأمر find للبحث داخل المجلد المحدد في المتغير LOG_DIR عن الملفات العادية فقط باستخدام الخيار -type f. الجزء الجوهري هنا هو المعامل -mtime +7 الذي يقوم بفلترة الملفات التي مر على آخر تعديل لها أكثر من سبعة أيام، مما يضمن الاحتفاظ بالسجلات الحديثة فقط وحذف ما تجاوز عمره أسبوعاً. بمجرد العثور على هذه الملفات القديمة، يقوم السكربت بتنفيذ الأمر rm -f عليها مباشرة عبر الخيار -exec لمسحها نهائياً دون طلب تأكيد من المستخدم، ثم يختتم العملية بطباعة كلمة "!Done" للإشارة إلى نجاح عملية التنظيف وتوفير المساحة.

رابط الاسكربت
Multi-Service Health Checker

يعمل هذا السكربت كأداة لمراقبة وإصلاح الخدمات (Service Monitoring and Self-healing) بشكل آلي، حيث يضمن بقاء الخدمات الحيوية للنظام قيد التشغيل دون تدخل بشري دائم.يبدأ الكود بتعريف مصفوفة services تضم قائمة بالخدمات المستهدفة (مثل خادم الويب وقواعد البيانات)، ثم يستخدم حلقة تكرار لفحص كل خدمة على حدة عبر الأمر systemctl is-active --quiet. السر البرمجي هنا يكمن في المتغير الخاص $? الذي يحفظ "حالة الخروج" (Exit Status) للأمر السابق؛ فإذا كانت القيمة لا تساوي صفراً (-ne 0) فهذا يعني أن الخدمة معطلة أو متوقفة. بمجرد اكتشاف تعطل الخدمة، يقوم السكربت بطباعة رسالة تحذيرية ثم ينفذ أمر إعادة التشغيل التلقائي sudo systemctl restart لإعادتها للعمل فوراً، بينما يكتفي بطباعة رسالة "OK" إذا كانت الخدمة تعمل بشكل سليم، مما يجعل هذا السكربت مثالياً لجدولته كمهام دورية (Cron Jobs) لضمان استقرار الخادم.

رابط الاسكربت
Network IP Scanner

يُعد هذا السكربت أداة ذكية لإجراء مسح للشبكة المحلية (IP Scanner) يدوياً، حيث يهدف لاكتشاف الأجهزة المتصلة والنشطة ضمن نطاق معين دون الحاجة لأدوات خارجية معقدة. يعتمد الكود على حلقة تكرار for تمر عبر الأرقام من 1 إلى 254 لبناء عنوان IP كامل لكل جهاز محتمل في الشبكة، ثم يستخدم أمر ping بإرسال حزمة واحدة فقط -c 1 مع مهلة زمنية قدرها ثانية واحدة -W 1. لضمان نظافة واجهة الأوامر، يتم توجيه مخرجات أمر الـ ping إلى "الثقب الأسود" في Linux وهو /dev/null. تكمن الفكرة الجوهرية في فحص حالة الخروج $?؛ فإذا كانت مساوية للصفر (-eq 0) فهذا يعني أن الجهاز استجاب للطلب وهو نشط حالياً، ليقوم السكربت عندها بطباعة عنوانه للمستخدم، مما يجعله وسيلة سريعة وفعالة لمراقبة الأجهزة المتصلة بشبكتك الخاصة.

رابط الاسكربت
Ping

يقدم هذا السكربت أسلوباً تفاعلياً لاختبار اتصال المضيف (Host Connectivity Test)، حيث يجمع بين إدخال البيانات من المستخدم ومعالجتها منطقياً لتحديد حالة الشبكة. تبدأ الدورة البرمجية بطلب عنوان IP من المستخدم وتخزينه في المتغير ip باستخدام الأمر read لتخصيص عملية الفحص، يليه تنفيذ أمر ping -c 4 الذي يرسل أربع حزم بيانات لاستطلاع استجابة الطرف الآخر. تكمن الذكاء في السكربت في استخدام المتغير $? لفحص حالة نجاح العملية؛ فإذا عادت القيمة بـ 0 (أي استلم النظام ردوداً صحيحة)، يطبع السكربت "Host is UP" كدليل على سلامة الاتصال. أما في حال فشل الوصول للمضيف أو انقطاع المهلة الزمنية، فإن السكربت ينتقل تلقائياً لخانة else ليطبع رسالة التنبيه "Host is DOWN"، مما يوفر للمسؤول عن النظام تقريراً فورياً ومبسطاً عن حالة الاتصال دون الحاجة لتحليل مخرجات الشبكة المعقدة يدوياً.

رابط الاسكربت
Port Scanner

يُعتبر هذا السكربت أداة ذكية لفحص حالة المنافذ (Port Scanning) باستخدام ميزات Bash الداخلية، دون الحاجة للاعتماد على أدوات خارجية مثل Telnet أو Netcat. تعتمد آلية العمل على استقبال عنوان IP ورقم المنفذ (Port) من المستخدم، ثم محاولة فتح اتصال عبر المسار الافتراضي /dev/tcp/$ip/$port؛ حيث يستخدم السكربت أمر timeout 1 لضمان عدم تعليق البرنامج في حال كان المنفذ لا يستجيب، مع توجيه أي أخطاء ناتجة إلى /dev/null للحفاظ على نظافة المخرجات. تتجلى الدقة البرمجية في فحص قيمة $?؛ فإذا نجح Bash في إنشاء الاتصال (قيمة 0)، يطبع السكربت "Port is OPEN" كإشارة إلى أن الخدمة نشطة، وفي حال الفشل أو انتهاء الوقت المخصص، يتم الانتقال لخيار else ليعلن أن المنفذ مغلق أو محمي بجدار ناري، مما يجعله وسيلة سريعة وفعالة لفحص الخدمات الشبكية يدوياً.

رابط الاسكربت
Privilege Escalation Enumeration Script

يُعد هذا السكربت أداة استطلاعية قوية تهدف إلى فحص احتمالات تصعيد الصلاحيات (Privilege Escalation) داخل نظام Linux، حيث يقوم بجمع معلومات حساسة قد يستغلها مدير النظام لسد الثغرات أو المختبر الأمني لاكتشاف نقاط الضعف. تبدأ عملية الفحص بتحديد الهوية الحالية للمستخدم عبر أمر whoami ومراجعة صلاحيات الـ SUDO الممنوحة له بواسطة sudo -l لمعرفة الأوامر التي يمكن تنفيذها بصلاحيات الجذر، يتبع ذلك البحث عن ملفات SUID الخطيرة باستخدام أمر find مع الفلتر -perm -4000؛ وهي ملفات تعمل بصلاحيات مالكها (غالباً الـ root) بغض النظر عن المستخدم الذي يقوم بتشغيلها، مما قد يمثل ثغرة أمنية كبيرة. كما يتفحص السكربت أذونات ملفات النظام الحساسة مثل /etc/passwd للتأكد من عدم وجود صلاحيات كتابة غير مصرح بها، ويختتم عمله بعرض أول 10 عمليات جارية في النظام باستخدام ps aux لإعطاء نظرة خاطفة على البرامج التي تعمل بخلفية النظام وصلاحياتها، مما يجعله مرجعاً سريعاً لتقييم الوضع الأمني العام للجهاز.

رابط الاسكربت
Reconnaissance (Recon) Script

يعمل هذا السكربت كأداة متكاملة للاستطلاع الرقمي (Reconnaissance)، حيث يقوم بجمع معلومات جوهرية عن أي هدف برمجياً لتسهيل عملية تحليل الشبكات أو اختبار الاختراق. تبدأ الرحلة بجلب عنوان IP الخاص بالهدف بطريقة ذكية عبر دمج أمر ping مع أدوات معالجة النصوص grep و awk لاستخلاص العنوان فقط، يتبعها استعلام DNS باستخدام nslookup لكشف سجلات النطاق وتوجيهاته. ثم ينتقل السكربت لمرحلة الكشف التقني عبر أداة nmap لفحص المنافذ الأكثر شيوعاً بشكل سريع، مما يعطي لمحة عن الخدمات النشطة على الخادم. وفي الختام، يستخدم السكربت أمر curl -I لجلب ترويسات بروتوكول HTTP (Headers)، وهي خطوة بالغة الأهمية لكشف نوع خادم الويب المستخدم، لغات البرمجة، والتقنيات الأمنية المفعلة، مما يوفر تقريراً أولياً شاملاً يختصر الكثير من البحث اليدوي.

رابط الاسكربت
Security Log Analyzer

يعمل هذا السكربت كأداة لتحليل السجلات الأمنية (Log Analysis)، حيث يركز بشكل محدد على رصد محاولات الاختراق أو الدخول الفاشلة للنظام عبر فحص ملف auth.log. تبدأ العملية باستخدام أمر grep لاستخراج كافة السطور التي تحتوي على عبارة "Failed password" وحفظها في ملف وسيط، يليه استخدام wc -l لإعطاء إحصائية سريعة حول العدد الإجمالي للمحاولات الفاشلة. تكمن القوة التحليلية في السطر الأخير؛ حيث يستخدم أداة awk لاستخلاص عناوين الـ IP الخاصة بالمهاجمين، ثم يقوم بترتيبها sort وعدّ تكرار كل منها uniq -c ليعرض في النهاية قائمة مرتبة تنازلياً بأكثر العناوين عدوانية تجاه النظام. تساعد هذه الطريقة مسؤولي النظام على تحديد مصادر الهجمات (Brute Force) بسرعة لاتخاذ إجراءات الحماية مثل حظر تلك العناوين باستخدام الجدار الناري.

رابط الاسكربت
Service Monitor & Auto-Restart Script

يقدم هذا السكربت آلية بسيطة وفعالة لمراقبة استمرارية الخدمة (Service Continuity)، حيث يهدف إلى ضمان بقاء خدمة معينة، مثل SSH، تعمل بشكل دائم دون انقطاع. يعتمد الكود على استخدام أداة إدارة النظام systemctl مع الخيار is-active للتحقق من الحالة الحالية للخدمة، مع إضافة --quiet لإخفاء المخرجات التقنية غير الضرورية والتركيز فقط على نتيجة الفحص. يكمن جوهر المنطق البرمجي في اختبار متغير الحالة $?؛ فإذا كانت قيمته لا تساوي الصفر، يستنتج السكربت فوراً أن الخدمة متوقفة ويقوم بتنفيذ أمر إعادة التشغيل sudo systemctl restart. بهذه الطريقة، يتحول السكربت إلى "حارس آلي" للنظام، حيث يطبع رسائل واضحة للمسؤول توضح ما إذا كانت الخدمة تعمل بسلام أو إذا كان قد تم التدخل لإصلاح عطل مفاجئ، مما يقلل من وقت توقف الخدمات الحيوية بشكل كبير.

رابط الاسكربت
System Monitoring Script

يعمل هذا السكربت كلوحة تحكم برمجية لمراقبة أداء النظام (System Monitoring)، حيث يجمع أهم البيانات الحيوية للجهاز ويعرضها بشكل منظم في مكان واحد. تبدأ العملية بكشف هوية المستخدم الحالي عبر whoami ومعرفة مدة تشغيل النظام دون انقطاع بواسطة أمر uptime. بعد ذلك، ينتقل السكربت لمراقبة الموارد الأساسية؛ فيستخدم أمر top لاستخلاص متوسط حمل المعالج (CPU Load)، وأمر free -h لعرض حالة الذاكرة العشوائية (RAM) بصيغة سهلة القراءة للبشر (مثل جيجابايت وميجابايت). يختتم السكربت جولته بفحص مساحة القرص الصلب باستخدام df -h لإظهار الأقسام المتاحة والمستخدمة، مما يجعله أداة مثالية للمسؤولين لمتابعة "صحة" الخادم بضغطة زر واحدة والتأكد من عدم وجود اختناقات في الموارد.

رابط الاسكربت
system commands

يعمل هذا السكربت الأخير كأداة استكشافية سريعة لجمع المعلومات الأساسية حول البيئة الحالية لنظام Linux بمجرد تشغيله. يستخدم السكربت ثلاثة أوامر جوهرية؛ الأول هو whoami لتحديد اسم المستخدم النشط، والثاني هو pwd (Print Working Directory) لعرض المسار الكامل للمجلد الحالي الذي يتواجد فيه المستخدم، بينما يقوم الأمر الثالث ls بسرد قائمة بكافة الملفات والمجلدات الموجودة في ذلك الموقع. تكمن أهمية هذا النوع من السكربتات في توفير "لقطة" فورية للسياق البرمجي، مما يساعد المطورين أو مديري الأنظمة على التأكد من موقعهم وصلاحياتهم قبل البدء بتنفيذ عمليات أكثر تعقيداً، وهو ما يمثل حجر الأساس في فهم التعامل مع واجهة السطر الأوامر.

رابط الاسكربت
JavaScript

هي لغة برمجة عالية المستوى، ديناميكية، ومفسرة، تُعد حجر الأساس لتطوير صفحات الويب التفاعلية . تعمل بشكل أساسي داخل متصفحات الويب لإضافة التفاعلية، مثل الرسوم المتحركة، التحديث الفوري للمحتوى، والتحقق من صحة النماذج. تمكنت من السيطرة على تطوير المواقع وتوسعت لتشمل تطوير خوادم الويب Node.js

تشغيل (JavaScript)

أسهل طريقة: افتح المتصفح Chrome اضغط F12 روح إلى Console واكتب:

        
         console.log("Hello JavaScript");

console: هو "كائن" (Object) برمجي يوفر وصولاً إلى أدوات التحكم والمتصفح البرمجية

.()log: هي "دالة" (Method) تُستخدم لطباعة أو عرض البيانات والمعلومات في لوحة التحكم (Console)

"Hello JavaScript": هو النص المراد عرضه، ويوضع بين علامات تنصيص لأنه يُعامل كـ "سلسلة نصية" (String)

الأقواس (): تُستخدم لتغليف البيانات التي نمررها للدالة لتقوم بمعالجتها وطباعتها

الفصلة المنقوطة ;: تُستخدم لإنهاء العبارة البرمجية، وهي علامة تخبر المحرك أن الأمر قد اكتمل هنا

المتغيرات (Variables)

تستخدم لتخزين البيانات

        
         let name = "Ali";
         let age = 20;

         console.log(name);
         console.log(age);

تعريف المتغيرات (let): قمت باستخدام الكلمة المحجوزة let لإنشاء "حاويات" في الذاكرة لتخزين البيانات

تخزين النصوص: المتغير name يحمل القيمة "Ali"، وهي سلسلة نصية (String) لأنها بين علامات تنصيص

تخزين الأرقام: المتغير age يحمل القيمة 20 كنوع عددي (Number)، ولا يحتاج لعلامات تنصيص

استدعاء القيم: عند كتابة console.log(name)، يبحث البرنامج عن القيمة المخزنة داخل الاسم name ويعرضها

النتيجة في الـ Console: سيظهر لك في لوحة التحكم الكلمة Ali في سطر، والرقم 20 في السطر الذي يليه

أنواع الإعلان:

        
          let x = 10;     // متغير عادي (يتغير)
          const y = 5;    // ثابت (لا يتغير)
          var z = 3;      // قديم (تجنبه)

let (الخيار الافتراضي): يُستخدم عندما تعلم أن قيمة المتغير ستتغير لاحقاً (مثل عداد في حلقة تكرار). يتميز بأنه يحترم "النطاق الكتلي" (Block Scope)، أي أنه يعيش فقط داخل الأقواس {} التي كُتب فيها.

const (الأكثر أماناً): يُستخدم للقيم التي لن تتغير أبداً. البرمجة الحديثة تنصح باستخدام const دائماً إلا إذا كنت متأكداً أنك ستحتاج لتغيير القيمة، فهذا يمنع الأخطاء غير المقصودة.

var (الموضة القديمة): كانت الطريقة الوحيدة قديماً، لكن مشكلتها أنها قد تسبب أخطاءً منطقية لأنها لا تحترم النطاق الكتلي (Function Scoped)، مما يجعل تتبع الأخطاء صعباً في المشاريع الكبيرة.

العمليات (Operators) 
        
         let a = 10;
         let b = 5;

         console.log(a + b); // جمع
         console.log(a - b); // طرح
         console.log(a * b); // ضرب
         console.log(a / b); // قسمة

الجمع (+): سيجمع القيمة 10 مع 5 والنتيجة 15

الطرح (-): سيطرح 5 من 10 والنتيجة 5

الضرب (*): سيضرب 10 في 5 والنتيجة 50

القسمة (/): سيقسم 10 على 5 والنتيجة 2

الشروط (If Statement) 
        
         let age = 18;

         if (age >= 18) {
         console.log("مسموح الدخول");
         } else {
         console.log("ممنوع الدخول");
         }

الشرط (if): نضع بين القوسين (age >= 18) عملية مقارنة. يسأل البرنامج: "هل القيمة الموجودة في age أكبر من أو تساوي 18؟"

القيمة المنطقية (Boolean): نتيجة هذا السؤال تكون دائماً إما true (نعم) أو false (لا)

كتلة التنفيذ { }:

إذا كان الشرط صحيحاً (true)، يتم تنفيذ الكود الموجود داخل أقواس الـ if فقط

إذا كان الشرط خاطئاً (false)، يتجاهل البرنامج القوس الأول تماماً وينتقل فوراً إلى ما بعد else

        
          let password = "1234";

          if (password === "1234") {
          console.log("تم تسجيل الدخول");
          } else {
          console.log("كلمة المرور خطأ");
          }

المقارنة المتطابقة (===): هذا المعامل لا يتحقق فقط من "القيمة" (أنها 1234)، بل يتحقق أيضاً من "النوع" (أنها نص String وليست رقماً Number)

بنية الشرط: بما أن المتغير password يحتوي تماماً على النص "1234"، سيتحقق الشرط وتُنفذ جملة الترحيب

لنتيجة: سيظهر لك في الـ Console: تم تسجيل الدخول

الحلقات (Loops)

for loop

        
         for (let i = 1; i <= 5; i++) {
             console.log(i);
         }

لبداية (let i = 1): هنا ننشئ "عداداً" ونبدأه من الرقم 1. يتم تنفيذ هذا الجزء مرة واحدة فقط في البداية

الشرط (i <= 5): قبل كل لفة، يسأل البرنامج: "هل i لا تزال أصغر من أو تساوي 5؟". إذا كان الجواب نعم، يستمر التكرار

التحديث (i++): بعد نهاية كل لفة، نزيد قيمة i بمقدار 1 (هذا ما تفعله ++)

كيف تسير العملية؟

اللفة 1: i هي 1، يطبع 1، ثم تصبح i هي 2

اللفة 2: i هي 2، يطبع 2، ثم تصبح i هي 3

... وهكذا حتى يصل إلى 5.

النهاية: عندما تصبح i هي 6، يفشل الشرط (6 <= 5 خطأ)، فيتوقف التكرار فوراً

        
          for (let i = 1; i <= 10; i++) {
              console.log("الرقم: " + i);
          }

النطاق: الحلقة ستكرر الكود 10 مرات (من 1 إلى 10)

عامل الجمع (+): عندما يُستخدم مع نص ورقم، فإنه لا يقوم بعملية حسابية، بل يقوم بـ "لصق" الرقم بجانب النص ليصبح جملة واحدة

النتيجة: سيظهر في لوحة التحكم (Console) قائمة مرتبة تبدأ بـ الرقم: 1 وتنتهي بـ الرقم: 10

while loop

تستخدم عندما لا تعرف عدد التكرارات مسبقًا

        
          let i = 0;

          while (i < 5) {
              console.log(i);
              i++;
          }

نقطة البداية (let i = 0): قمت بتعريف العداد خارج الحلقة

الشرط (i < 5): قبل كل دخول للحلقة، يسأل البرنامج: "هل i أقل من 5؟"

جسم الحلقة: يتم طباعة قيمة i الحالية

تحديث العداد (i++): هذا السطر هو الأهم؛ بدونه ستدخل في "حلقة لا نهائية" (Infinite Loop) لأن i ستبقى دائماً 0، مما قد يؤدي لتعليق المتصفح

اسكربتات بلغة جافا سكربت

File Integrity Monitor

يعمل هذا السكريبت كأداة مراقبة لسلامة البيانات من خلال استدعاء مكتبة `fs` البرمجية لمراقبة التغييرات التي تطرأ على ملف الإعدادات `config.json` بشكل مستمر ولحظي. بمجرد اكتشاف أي تعديل في محتوى الملف، يقوم الكود تلقائياً بتنفيذ دالة برمجية تستخرج طابع الوقت الدقيق لحالة الملف الجديدة وتقارنها بالحالة السابقة. من منظور الأمن السيبراني، يمثل هذا الكود نظام إنذار مبكر ضد الاختراق؛ حيث يطبع تنبيهاً فورياً في شاشة التحكم يكشف عن أي محاولة عبث أو تعديل غير مصرح به. تكمن قوة هذا السكريبت في توثيق الحوادث رقمياً عبر خاصية `mtime` التي تساعد في التحقيقات الجنائية لتحديد وقت وقوع الهجوم بدقة. وبذلك، يتحول هذا السطر البرمجي البسيط إلى حارس رقمي يضمن بقاء ملفات النظام الحساسة تحت الرقابة المستمرة لتعزيز استراتيجية الدفاع النشط.

رابط الاسكربت
URL Defang

يعمل هذا السكريبت كأداة أمنية تقنية تُعرف بـ **"إبطال مفعول الروابط" (URL Defanging)**، حيث يستخدم تعابير برمجية نمطية (Regex) لتعديل بنية الروابط الخبيثة ومنع النقر عليها بالخطأ. يقوم الكود باستبدال النقاط الحساسة في الرابط بوضعها داخل أقواس `[.]` وتغيير بروتوكول `http` إلى `hxxp` لتحويل الرابط من عنصر تفاعلي نشط إلى مجرد نص غير قابل للضغط. من منظور الأمن السيبراني، تكمن أهمية هذا الإجراء في حماية المحللين الأمنيين والمستخدمين أثناء تبادل معلومات التهديدات (Threat Intelligence)، حيث يمنع المتصفحات والبرامج من تحميل المواقع الضارة تلقائياً. وبذلك، يساهم السكريبت في تقليل مخاطر العدوى البرمجية عند تحليل الروابط المشبوهة داخل التقارير الأمنية، محولاً الرابط "المفترس" تقنياً إلى نسخة آمنة للعرض فقط.

رابط الاسكربت
Extracting IP addresses from logs

يؤدي هذا السكريبت دوراً محورياً في عمليات "تحليل سجلات النظام" (Log Analysis)، حيث يستخدم مكتبة `fs` لقراءة محتوى ملف السجلات `access.log` بالكامل ومعالجته برمجياً. يعتمد الكود على تعبير نمطي (Regex) دقيق مصمم خصيصاً لاكتشاف استخراج عناوين الـ IP من بين آلاف السطور النصية المزدحمة بالبيانات. من منظور الأمن السيبراني، يمثل هذا السكريبت أداة استخبارات تقنية تساعد المحللين في حصر جميع العناوين التي حاولت الوصول للنظام، مع تصفية المكرر منها باستخدام خاصية `new Set` لإنتاج قائمة نقية بالجهات المتصلة. وتتجلى أهمية هذه العملية في كشف محاولات الهجمات الموزعة (DDoS) أو رصد عمليات مسح المنافذ (Port Scanning)، مما يسمح لمسؤولي الأمن بتحديد العناوين المشبوهة وحظرها قبل تفاقم الهجوم. وبذلك، يحول السكريبت البيانات الخام الضخمة إلى معلومات استخباراتية منظمة تسهم بشكل مباشر في تعزيز جدار الحماية الرقمي للمنشأة.

رابط الاسكربت
Brute Force

يجسد هذا السكريبت آلية دفاعية برمجية تُعرف بـ **"كشف محاولات التخمين" (Brute-Force Detection)**، حيث يقوم بتتبع عدد محاولات تسجيل الدخول الفاشلة لكل عنوان IP على حدة وحفظها داخل كائن برمجي. يعمل الكود من خلال دالة تراكمية تزيد عداد الأخطاء في كل مرة يفشل فيها المستخدم في الدخول، وعند تجاوز العتبة المحددة (أكثر من 5 محاولات)، يطلق السكريبت إنذاراً أمنياً فورياً يحذر من محاولة اختراق محتملة. من منظور الأمن السيبراني، يعد هذا السكريبت خط الدفاع الأول ضد هجمات القوة الغاشمة التي تستهدف كسر كلمات المرور عبر التجربة والخطأ، مما يسمح للنظام باتخاذ إجراءات وقائية لاحقة مثل حظر العنوان المشبوه مؤقتاً. تكمن أهمية هذا الأسلوب في القدرة على التمييز بين الخطأ البشري الطبيعي وبين السلوك الآلي الممنهج الذي تتبعه برمجيات الاختراق. وبذلك، يوفر السكريبت طبقة أمان حيوية تحمي حسابات المستخدمين من الوصول غير المصرح به وتمنع استنزاف موارد الخادم في عمليات التحقق المتكررة.

رابط الاسكربت
Password strength check

يُطبق هذا السكريبت معياراً برمجياً صارماً للتحقق من **"قوة كلمات المرور" (Password Complexity Validation)**، حيث يستخدم تعبيراً نمطياً (Regex) معقداً لضمان استيفاء كلمة المرور لشروط أمنية دقيقة. يعمل الكود عبر سلسلة من "الفحوصات الاستشرافية" التي تفرض وجود أحرف صغيرة، وأحرف كبيرة، وأرقام، ورموز خاصة، بالإضافة إلى شرط الطول الأدنى الذي لا يقل عن 12 حرفاً. من منظور الأمن السيبراني، يمثل هذا السكريبت أداة وقائية حاسمة تمنع المستخدمين من إنشاء حسابات بكلمات مرور سهلة التوقع، مما يرفع بشكل كبير من "التكلفة الحسابية" لأي هجوم تخمين أو استخدام لقواميس كلمات المرور الجاهزة. وتتجلى أهمية هذا النهج في حماية الهوية الرقمية ومنع اختراق الحسابات عبر تقنيات الهندسة الاجتماعية أو القوة الغاشمة، حيث تظل كلمات المرور الطويلة والمتنوعة هي الحصن الأول ضد محاولات الوصول غير المشروع. وبذلك، يضمن السكريبت تطبيق سياسات أمنية قياسية تحول دون ضعف السلسلة الأمنية للنظام عند نقطة دخول المستخدم.

رابط الاسكربت
Simple Port Scanner

يعمل هذا السكريبت كأداة استطلاع تقنية تُعرف بـ **"ماسح المنافذ" (Port Scanner)**، حيث يستخدم مكتبة `net` لإنشاء اتصالات شبكية تجريبية عبر بروتوكول TCP مع عناوين محددة. يقوم الكود بمحاولة فتح "سوكت" (Socket) على قائمة من المنافذ الشهيرة (مثل 80 و443) والتحقق مما إذا كانت "مفتوحة" وتستجيب للطلبات خلال مهلة زمنية قصيرة. من منظور الأمن السيبراني، يمثل هذا السكريبت أداة مزدوجة الاستخدام؛ فهي تساعد مسؤولي النظام في "تدقيق الشبكة" (Network Auditing) للتأكد من إغلاق المنافذ غير الضرورية، بينما يستخدمها المهاجمون في مرحلة "جمع المعلومات" لاكتشاف الخدمات النشطة والثغرات المحتملة. وتبرز أهمية هذا الإجراء في كشف الأبواب الخلفية أو الخدمات الضعيفة التي قد تكون مدخلاً للاختراق، مما يسمح بتقليص "سطح الهجوم" (Attack Surface) للمنشأة. وبذلك، يحول السكريبت عملية فحص الشبكة المعقدة إلى مهمة مؤتمتة وسريعة تمنح رؤية واضحة حول مدى تحصين الجهاز أو الخادم.

رابط الاسكربت
HTTP Banner Grabbing

يُنفذ هذا السكريبت تقنية أمنية تُعرف بـ **"بصمة الخادم" (Banner Grabbing)**، حيث يستخدم مكتبة `http` لإرسال طلب استعلام إلى موقع معين بهدف استخراج معلومات من "رأس الاستجابة" (HTTP Response Headers). يركز الكود تحديداً على جلب قيمة العنوان `server` الذي يكشف غالباً عن نوع وإصدار برمجيات الخادم المستخدمة (مثل Apache أو Nginx). من منظور الأمن السيبراني، تعتبر هذه العملية الخطوة الأولى في مرحلة "الاستطلاع" (Reconnaissance)، حيث تساعد المحللين على تحديد الأهداف ومعرفة ما إذا كان الخادم يعمل بإصدار قديم يحتوي على ثغرات معروفة. وتتجلى أهمية هذا الإجراء في "إدارة الثغرات"، حيث يُنصح أمنياً بإخفاء هذه المعلومات أو تمويهها لتقليل البيانات المتاحة للمهاجمين حول بنية النظام التحتية. وبذلك، يحول السكريبت طلب اتصال بسيط إلى أداة لكشف هوية التقنيات المشغلة للمواقع، مما يساهم في تقييم مستوى الأمان الخارجي للمنظمة.

رابط الاسكربت
Base64 (Data Exfiltration Simulation)

يقوم هذا السكريبت بعملية **"ترميز البيانات" (Base64 Encoding)**، حيث يستخدم كائن `Buffer` المدمج في Node.js لتحويل النصوص العادية إلى صيغة نصية تعتمد على نظام 64 حرفاً لضمان نقلها بشكل آمن عبر الأنظمة المختلفة. من منظور الأمن السيبراني، يمثل هذا الكود وسيلة لتمويه البيانات وتجاوز مرشحات النصوص التي قد تمنع إرسال الرموز الخاصة، لكنه **لا يعتبر تشفيراً** لأنه يمكن فك ترميزه بسهولة تامة ولا يوفر أي خصوصية حقيقية للبيانات الحساسة. تبرز أهمية فهم هذا التمييز في منع الوقوع في خطأ أمني فادح وهو الاعتماد على "الترميز" بدلاً من "التشفير القوي" لحماية أسرار المستخدمين. وبذلك، يعمل السكريبت كأداة لتحويل صيغ البيانات لتسهيل معالجتها برمجياً، مع التذكير الدائم بضرورة استخدام خوارزميات تشفير حقيقية مثل AES عند التعامل مع بيانات حساسة فعلياً.

رابط الاسكربت
(Simple Directory Brute-forcer)

ينفذ هذا السكريبت تقنية استطلاع هجومية تُعرف بـ **"اكتشاف المسارات المخفية" (Directory Brute Forcing)**، حيث يستخدم مكتبة `axios` لإرسال طلبات HTTP متتالية إلى موقع مستهدف لاختبار وجود مجلدات حساسة. يعمل الكود عبر محاولة الوصول إلى قائمة من المسارات الشائعة (مثل `admin` و `backup`) والتحقق من استجابة الخادم؛ فإذا كان رمز الحالة هو **200**، فهذا يعني أن المسار متاح وقابل للوصول. من منظور الأمن السيبراني، تكمن خطورة هذا الإجراء في كشف لوحات التحكم أو ملفات النسخ الاحتياطي التي قد لا تكون محمية بشكل كافٍ، مما يمنح المهاجم موطئ قدم داخل النظام. وتبرز أهمية هذا الفحص في مرحلة "البحث عن الثغرات" لتقييم مدى تعرض سطح الويب للمخاطر، مع التوصية الأمنية بضرورة تعطيل خاصية عرض المجلدات واستخدام جدران حماية تطبيقات الويب (WAF) لحظر مثل هذه المحاولات التكرارية. وبذلك، يحول السكريبت عملية البحث اليدوي المجهدة إلى أداة سريعة لاختبار متانة التكوينات الأمنية للمواقع.

Extracting links from a web page

ينفذ هذا السكريبت تقنية أتمتة تُعرف بـ **"الزحف للروابط" (Web Crawling/Link Extraction)**، حيث يستخدم مكتبة `axios` لجلب كامل الكود المصدري (HTML) لصفحة ويب معينة ومعالجتها برمجياً. يعتمد الكود على تعبير نمطي (Regex) مصمم خصيصاً للبحث عن وسم `href` واستخراج كافة الروابط التشعبية الموجودة داخل الصفحة. من منظور الأمن السيبراني، يمثل هذا السكريبت أداة أساسية في مرحلة "رسم خرائط الأهداف" (Target Mapping)، حيث يساعد المحللين على فهم بنية الموقع، واكتشاف الصفحات الخارجية المرتبطة به، أو تحديد نطاقات فرعية قد تكون هدفاً للفحص. وتتجلى أهمية هذه العملية في كشف روابط "التصيد الاحتيالي" أو الروابط الخبيثة التي قد تكون مخفية داخل كود الصفحة، مما يسهل عملية تحليل التهديدات وتوسيع نطاق الاختبار الأمني للموقع. وبذلك، يحول السكريبت عملية تتبع الروابط اليدوية إلى مهمة برمجية سريعة تمنح رؤية شاملة حول الاتصالات الخارجية والداخلية لأي موقع مستهدف.
